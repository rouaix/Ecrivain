Téléchargez le zip du handoff via la carte ci-dessus. Voici comment l'utiliser dans Claude Code :
1. Décompresser dans votre repo Écrivain

unzip design_handoff_ecrivain_project_page.zip -d ~/code/ecrivain/
cd ~/code/ecrivain/

2. Lancer Claude Code à la racine du projet

claude

3. Pointer Claude vers le handoff

Premier message à coller dans Claude Code :

    Lis design_handoff_ecrivain_project_page/README.md et les 4 fichiers direction-*.jsx dans source/. C'est le redesign de la page projet (/ecrivain/project/{id}). Avant d'écrire du code : explore mon codebase pour comprendre la stack (framework, composants existants, conventions), puis propose-moi un plan d'implémentation des 4 thèmes A/B/C/D comme un sélecteur dans les paramètres utilisateur. N'écris rien tant que je n'ai pas validé le plan.

4. Quelques bons prompts de suivi

    "Implémente d'abord la direction A (Manuscrit) sur la page projet, en gardant les data réels de la BDD. Pas de mock."
    "Ajoute un sélecteur de thème dans /settings, persiste en BDD sur user.theme."
    "Porte la direction B en réutilisant les composants <ChapterRow>, <ProgressCard> que tu viens de créer — ne duplique pas la structure."

5. Conseils

    Une direction à la fois. Faire A en entier, valider, puis B/C/D — sinon Claude Code va paralléliser et casser.
    Garde les jsx du handoff sous les yeux : tokens, espacements et hiérarchie sont précis dans le README.
    Polices : EB Garamond, Lora, Cormorant Garamond, Source Serif 4 + Inter doivent être chargées conditionnellement selon le thème actif.
    Le design-canvas.jsx dans source/ est juste le wrapper de présentation — à ignorer côté implémentation.

Si Claude Code n'a pas encore accès au repo, il faudra cloner d'abord et lancer claude dans le bon dossier.
